Hexo博客搭建与插件改造完全指南

发表于 2017-01-12 更新于 2026-03-21
从零开始搭建Hexo静态博客，并记录所有插件改造过程

经过多次尝试，终于搭建好了自己的博客！本文记录了完整的搭建过程和后期插件改造，让博客更加好用。

一、为什么选择 Hexo

以前用过 WordPress，折腾各种主题和插件，还要维护服务器，后来发现很多技术大牛都把博客放在 GitHub Pages 上——静态博客，无需管理后台，用 Markdown 写文章，简直是程序员的最佳选择！

二、搭建及改造流程概览

只需要 5 步就能完成搭建：

  1. 安装 Node.js - Hexo 基于 Node.js，需要先安装
  2. 安装 Hexo - 用 npm 安装 Hexo 博客框架
  3. 配置 SSH - 让部署时不需要每次输入密码
  4. 部署上线 - 推送到 GitHub Pages
  5. 插件改造 - 选择符合自己使用习惯的插件安装并优化hexo博客

3.3 配置 Git SSH 密钥

为了让 Hexo 部署时不需要每次输入密码，需要配置 SSH 密钥。

第一步：配置 Git 用户信息

第二步：生成 SSH 密钥

一路回车，会在 ~/.ssh 目录下生成两个文件：

  • id_rsa.github - 私钥（自己保管）
  • id_rsa.github.pub - 公钥（放到 GitHub）

第三步：添加到 GitHub

  1. 复制公钥内容：cat ~/.ssh/id_rsa.github.pub
  2. 打开 GitHub → Settings → SSH and GPG keys → New SSH key
  3. 粘贴公钥，保存
